Bukkit 及其衍生端（如 Spigot、Paper、Purpur 等），可以安装插件，如果你想玩生存，选这个就可以。（还可以安装插件，比如登录插件、粘液科技等等。这几个衍生端里，最推荐使用 Purpur。）

Fabric 端，可以安装 Fabric 模组，玩生电必选这个，因为 Fabric 没有修改原版特性，并且生电的辅助模组（如 Tweakeroo，MiniHUD，Carpet 等）多多少少都需要服务端也安装这些模组才能正常使用。

Forge 端，可以安装 Forge 模组，如果想玩整合包或者模组生存，选这个就对了。（注意区分 Forge 模组和 Fabric 模组，别下错版本了）

如果你下载了网上的地图想和朋友联机，推荐直接使用 Minecraft 原版服务端，因为 Bukkit 端某些版本的存档保存方式与传统方式不一样，可能地图无法正常加载。（Dim-1 和 Dim1 从 world 文件夹单独分到 world_nether 和 world_the_end 文件夹了）

# 多人游戏的运作

如果你经常玩 Minecraft 多人游戏，你可能会问一个问题：为什么我每次都要输入一个域名呢？

域名被解析到一个 IP 地址上，一个 IP 地址代表了一个服务器，服务器也是一台计算机。

在你（玩家）的电脑上运作的那个叫做客户端，而那个远在几千公里之外的计算机…… 就叫服务器。

服务器是在多人游戏中用来交换玩家间数据的。客户端把数据发送给服务器，服务器把数据发送给其它客户端。

当一个 Minecraft 服务器开始运转时，那里生成了一个世界。

客户端加入游戏时，将世界的相关信息下载下来（相当于复制一份），并在本地运转起来。

当服务端或客户端发生了什么事情时，客户端（或服务端）会使用一种名为数据包的东西传递信息，收到数据包的一方根据其中的内容，更新自己的那个世界。

比方说，你挖掉了一个方块，客户端就会给服务端发送一个数据包，告诉服务端你挖掉了一个方块。服务端接收到这个数据包之后，会将你挖掉的方块移除并生成掉落物。此时，挖掉方块和掉落物的生成也会以数据包的形式发送到客户端，客户端接收到之后进行画面渲染，最后变成屏幕上显示的内容。

在单人游戏，也会有一个内置服务端，只不过你看不到。然而如果你开的是服务器，服务端一般会以 CMD 窗口展现。

# 三种服务端

# Bukkit 服务端

Mojang 制造的 Minecraft 原版服务端，可以把它想象成一个盒子，你在外面是不知道它怎么工作的。

后来一个叫 Bukkit 的人看不下去了，在盒子上面戳了一个洞，允许开发者把一些东西塞进去。这些东西就是插件，你可以自己编写内容塞进 mc 里。

后来 Mojang 发现不对啊，Bukkit 这家伙怎么拿着我的东西在那展览呢？

Mojang 越想越不对劲，于是拿着 EULA（最终用户许可条款）和 DCMA（数字千年版权法案）把 Bukkit 告上了法庭。

Bukkit 败诉，项目被迫停止。

Bukkit 停止后，开发者们当然不乐意啦。开什么玩笑？之前辛辛苦苦写的插件不能用了？

然而生气归生气，DCMA 是没办法挑战的，于是开发者们只好无比失望地洗洗睡了……

后来有一天，Bukkit 的一些开发人员想到了办法。

既然我不能把盒子打洞之后给你，那我就把打洞的方法教给你嘛！

开发人员们想到了新的办法：提供一个不包含 Minecraft 源代码的部署工具，里面记载了修改服务端的方法。

要使用时，这个工具会在服主的计算机上下载一份 Minecraft 客户端，完成反编译、注入、重新编译、打包等工作。就像一个安装程序一样。你可能见到过的 BuildTools.jar 就是它。

由于以 Bukkit 为名继续项目可能不方便，开发人员们创立了新的服务端：Spigot。

Spigot 看着 Bukkit 的图纸，又在盒子上打了个洞，和原来的 Bukkit API 一起，放入 BuildTools.jar 中。这个加上去的 API，就是 Spigot API。

Paper 觉得 Spigot 的运转速度太慢了，于是拿着 Spigot 的盒子又打了几个洞，还把盒子加强了一番。这就是 Paper 服务端。

这就是 Bukkit 服务端、Spigot 服务端和 Paper 服务端。它们都有一个共同的特点：可以安装插件 (Plugin)。你们熟知的 Hypixel 服务器里各种功能的实现，都是插件实现的 (当然也有修改服务端)。

# Fabric 服务端

看到 Fabric，你第一反应是不是模组加载器？没错，Fabric 服务端其实就是能安装 Fabric 模组的服务端。

# Forge 服务端

同上，这是一个能安装 Forge 模组的服务端。

# 服务端的选择

那么最关键的问题来了。这三个服务端有什么区别？该如何选择？

首先来说 Fabric 服务端。如果你想玩原版纯净生存，或者生电生存，那么首选 Fabric 端。因为这本来就是一个为原版服务的端，Fabric 模组也大多数都是如此。

再来说 Forge 服务端。选择该服务端只有一种情况，就是游玩 Forge 模组的整合包时需要和朋友联机，就用 Forge 服务端。
我极不推荐拿 Forge 服务端玩原版。因为 forge 修改了太多内容，导致你玩的 mc 和原版有很多特性的差异。

最后就是 Bukkit 服务端。这个服务端用途非常广泛。因为你安装插件只需要在服务器安装就行了，而玩家不需要安装任何东西，原版就能进入。并且，Bukkit 也有很多很多优质插件，比如 essential 管理插件等等。如果你想实现自己的功能，还可以亲自编写插件来实现。所以这是一个用途非常广泛的服务端。如果 fabric 和 forge 都不适合你，那就用 Bukkit 服务端吧。
这时，肯定有人会问：Bukkit 服务端也有 spigot、paper 等等，该如何选择呢？
这里就直接推荐 paper 及其衍生端了。衍生端包括 purpur 等，都是非常不错的服务端。
